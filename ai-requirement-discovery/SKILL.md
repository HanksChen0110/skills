# AI产品需求挖掘

> 工作流位置：**需求挖掘** → 用户调研 → 需求分析 → 竞品分析 → 技术选型 → 数据处理 → AI PRD → 评估体系 → Badcase治理

## 核心流程

```
1. 识别需求来源（内部战略/外部用户/数据驱动）
2. 问题框架分析（差距感知/系统透视/矛盾动力学）
3. 需求池录入（背景/价值/状态/方案）
4. 优先级排序（KANO模型 + 四象限法）
5. AI适用性初判（是否真的需要AI？）
```

## KANO 模型速查

| 类型 | 定义 | AI产品示例 |
|------|------|-----------|
| 必备型(M) | 不满足则极不满 | 回答准确、无幻觉 |
| 期望型(O) | 满足则满意 | 回答快速、内容相关 |
| 魅力型(A) | 超预期惊喜 | 主动追问澄清意图 |
| 无差异型(I) | 有无均不影响 | 装饰性功能 |
| 反向型(R) | 满足反而降低满意度 | 强制弹窗 |

## AI 场景适用性快速判断

```
高泛化 + 数据充足 + 有容错机制 → 适合用AI
强确定性 + 低容错 + 成本敏感 → 优先规则/传统方案
```

**4个关键问题：**
1. 传统技术能否解决？
2. 数据量是否足够？
3. 对错误的容忍度如何？
4. 成本预算是否支撑？

## 需求池核心字段

```
基本信息：模块/业务线/来源/优先级(P0-P2)/状态
需求内容：背景(为什么做) + 价值(解决后的收益) + 验收标准
方案初判：技术可行性 + 预估工期
```

- 需求优先级矩阵
- 商业需求文档（BRD）草稿
