# 功能设计技能

使用本技能把需求分析结果整理为可评审的功能设计说明。

## 核心目标

- 把需求转成可交付的功能方案。
- 在需规前先明确功能边界、规则、状态、角色和异常处理。
- 为页面蓝图、用户故事、验收标准和需规提供中间层输入。

## 默认内容

- 功能目标
- 功能拆解
- 业务规则
- 状态流转
- 异常场景
- 角色与权限
- 接口与数据项约束
- 与上下游模块关系

## 工作流

1. 先读取需求分析、用户故事或旧稿。
2. 拆分一级功能和关键子功能。
3. 整理规则、状态、异常和权限。
4. 明确接口与数据约束。
5. 输出功能设计说明或评审稿。

## 输出建议

- 功能设计说明
- 功能分解表
- 规则清单
- 状态流转说明
- 异常场景说明

## 禁止事项

- 不得跳过核心规则和异常场景。
- 不得把功能设计写成页面原型说明。
- 不得把未确认规则写成最终定论。
