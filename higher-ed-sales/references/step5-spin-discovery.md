## 核心原理

现场拜访是整个销售流程中**最重要的环节**。核心原则：

1. **不要一上来就介绍产品**——先探索痛点，再针对性展示方案
2. **Implication问题（揭示后果）比Need-payoff问题（描绘价值）更能激活行动意愿**——因为损失厌恶是人类最强的心理驱动力
3. **Situation问题能自己查到的就不要问**——问了反而显得没做准备

## SPIN问题框架详解

### Situation（现状问题）— 尽量跳过

**目的**：建立背景认知。
**原则**：步骤2已收集的信息不要重复问，只确认少量关键信息。
**限制**：不超过2-3个，多了买家会觉得你没做准备。

**示例**：
- "今年学校在数字资源建设方面有什么新的规划方向？"
- "目前学校教材选用流程是由教务处统筹还是各学院自主？"

### Problem（痛点问题）— 打开缺口

**目的**：让客户自己说出不满和困难。
**原则**：开放式提问，引导客户描述具体困难。

**示例**：
- "目前各二级学院在教材选用过程中，遇到的最普遍的问题是什么？"
- "老师们对现有教材配套的数字化资源满意度如何？"
- "学校在申报精品课程/教学资源库时，最大的瓶颈在哪里？"
- "现有教材的更新频率能否跟上行业技术发展的速度？"

### Implication（暗示问题）— 核心发力点

**目的**：揭示问题不解决的**后果**，激活损失厌恶。
**原则**：这是SPIN中最重要的一环。买家会为避免损失而行动，而非为追求收益而行动。

**示例**：
- "如果实训教材缺乏配套的视频/动画资源，对学生技能掌握和考证通过率会有什么影响？"
- "如果明年学校要接受教学评估/专业认证，现有的教材和资源储备能否满足要求？"
- "如果数字资源建设滞后，对学校在'双高计划'验收或新一轮申报中会产生什么影响？"
- "如果老师们花大量时间自己制作教学资源，对教学科研精力分配会有什么影响？"
- "如果教材配套资源不完善，学生课前预习和课后复习的效果会打多少折扣？"

### Need-payoff（需求回报问题）— 收尾引导

**目的**：让客户自己描绘解决方案的价值。
**原则**：在Implication问题建立了紧迫感之后，用Need-payoff让客户看到希望。

**示例**：
- "如果能有一套与教材深度配套的数字化资源（视频+动画+H5互动），对老师们的教学效率和学生的学习效果会有多大提升？"
- "如果有一站式的数字教材出版服务，能帮学校在申报精品课程时节省多少时间？"
- "如果现有教材的数字化资源覆盖率从30%提升到90%，对学校的教学质量评价会产生什么积极影响？"

## 高职教育行业SPIN问题库

### 纸质教材场景

| 阶段 | 问题 |
|------|------|
| Problem | "老师们反映现有教材最大的不足是什么？内容陈旧？案例不够贴近行业？还是配套资源不足？" |
| Problem | "教材选用过程中，老师们是倾向于选用知名出版社还是更看重内容适配度？" |
| Implication | "如果教材内容与行业最新技术脱节，对学生实习就业和技能考证会产生什么影响？" |
| Implication | "如果教材选用过程中频繁更换，对教学连续性和学生体验会有什么影响？" |
| Need-payoff | "如果能提供与行业企业联合开发的教材，内容每年更新，对教学质量和学生就业率会有多大提升？" |

### 数字资源制作场景

| 阶段 | 问题 |
|------|------|
| Problem | "学校目前制作数字资源（视频、动画、H5）主要依靠老师自己还是外包？最大的困难是什么？" |
| Problem | "已建成的数字资源，老师们的实际使用率如何？存在什么问题？" |
| Implication | "如果学校投入大量资金建设的数字资源使用率很低，对后续争取专项经费会有什么影响？" |
| Implication | "如果各二级学院各自为政建设资源，无法共享复用，会造成多少重复投入？" |
| Need-payoff | "如果能有一支专业团队提供从策划到制作的一站式服务，并且资源能跨专业共享，对学校的投入产出比会有多大改善？" |

### 数字教材出版场景

| 阶段 | 问题 |
|------|------|
| Problem | "学校目前有数字化教材的需求吗？主要障碍是什么？技术？资金？还是版权？" |
| Problem | "老师们对现有数字化教材平台的使用体验如何？满意和不满意的点分别是什么？" |
| Implication | "如果学校在数字化教材方面落后于同类院校，在教学质量评价和专业认证中会处于什么位置？" |
| Need-payoff | "如果能将现有纸质教材快速转化为数字教材，并且配套完整的互动资源，对学校的教学改革和品牌提升会有多大帮助？" |

## 拜访结构设计

### 标准拜访结构（约45分钟）

| 阶段 | 时间 | 做什么 | 关键原则 |
|------|------|--------|---------|
| 开场 | 5分钟 | 简短自我介绍 + 拜访目的说明 + 议程确认 | **不要一上来放PPT** |
| 需求探索 | 20-30分钟 | SPIN问题框架（重点在Problem和Implication） | 多听少说，7:3比例 |
| 价值传递 | 15-20分钟 | 针对探索中发现的1-2个核心痛点，展示方案和案例 | 包含量化结果 |
| 关系推进 | 5-10分钟 | 请求引荐其他关键角色，确认下一步行动计划 | 扩展关系线 |

### 开场话术模板

```
"X院长/主任您好，我是XX公司的XX。感谢您抽出时间。

今天拜访主要有三个目的：
第一，了解贵校在XX方面的现状和需求；
第二，分享一些我们在同类院校的成功经验，看看是否有借鉴价值；
第三，探讨是否有合作的可能性。

大概占用您40分钟左右时间，您看可以吗？"
```

### 关系推进话术模板

```
"今天跟您交流收获很大。您提到XX问题确实很关键。

我想请教一下，在教材/数字资源方面，学校还有哪些部门的领导需要了解这些信息？
您方便帮我引荐一下吗？"

"好的，那我这周内给您发一份针对贵校XX专业的初步方案，您看下周五方便再沟通一次吗？"
```

## 价值传递策略

### 核心原则

**Demo不是产品导览，而是"让买家看到自己的问题被实时解决"。**

### 结构

1. **量化问题**（30秒）：重述客户痛点，用具体数据
   > "您刚才提到，学校实训视频资源缺口大约200个，老师们自己制作一个视频平均要花3天时间。"

2. **展示方案**（10分钟）：针对1-2个核心痛点，展示对应解决方案
   > "我们为XX职业技术学院的机电专业制作了150个实训视频，覆盖XX个核心技能点。请看这个示例..."

3. **呈现结果**（5分钟）：用同类型学校的成功案例说话，包含量化结果
   > "使用我们的资源后，XX职业技术学院的学生技能考核通过率从72%提升到89%，教师备课时间减少了40%。"

### 案例呈现模板

```
学校：XX职业技术学院（同类学校、同类专业）
挑战：（与客户痛点相似的问题）
方案：（我们做了什么）
成果：（量化结果 — 用数据说话）
  - 学生技能考核通过率：72% → 89%
  - 教师备课时间：减少40%
  - 教学满意度：从68%提升到93%
```
