## 核心原理

在拜访客户之前，完成学校全景画像和关键人员识别。目标：
1. 把有限的面对面时间用在**最有价值的需求探索**上，而不是问买家觉得"你应该已经知道"的基础问题
2. 提前识别决策链条中的关键角色，为多线关系经营打下基础

## 学校全景画像模板

### 基础信息

| 维度 | 采集内容 | 信息来源 |
|------|---------|---------|
| 建校历史 | 成立时间、办学层次变迁 | 学校官网"学校简介" |
| 办学定位 | 应用型/技术型/综合型、行业背景 | 学校官网 |
| 在校生规模 | 全日制在校生人数 | 学校官网/质量年报 |
| 师资规模 | 专任教师数量、高级职称占比 | 学校官网/质量年报 |
| 占地面积 | 校区数量、总面积 | 学校官网 |

### 专业布局

| 维度 | 采集内容 | 分析要点 |
|------|---------|---------|
| 二级学院设置 | 各学院名称、成立时间 | 哪些是新设学院（有新增需求） |
| 重点专业 | 国家级/省级重点专业、特色专业 | 与公司产品线的匹配度 |
| 专业数量 | 开设专业总数 | 学校规模和资源投入能力 |
| 产教融合 | 合作企业、产业学院 | 了解学校的产业合作偏好 |

### 荣誉与资质

| 维度 | 采集内容 | 销售启示 |
|------|---------|---------|
| 双高计划 | 是否入选、哪个层次（A档/B档/C档）、哪些专业群 | 伴随专项资金，有数字资源建设需求 |
| 示范校/优质校 | 是否获评 | 有专项建设经费 |
| 其他荣誉 | 教学质量优秀奖、就业先进单位等 | 了解学校关注方向 |

## 采购历史分析方法

### 数据采集

从招标网站导出目标学校近3年采购数据，按以下维度整理：

| 项目类型 | 关注内容 |
|---------|---------|
| 教材采购 | 中标单位、金额、采购方式、采购节奏（几月招标） |
| 数字资源建设 | 教学资源库、精品课程、在线课程平台等项目 |
| 信息化项目 | 智慧校园、数字化教学平台、实训室建设等 |
| 其他相关 | 产教融合项目、社会服务项目等 |

### 投入比分析

```
教材采购额 : 数字资源采购额 : 信息化总投入

示例分析：
- 比例为 3:1:5 → 学校重视信息化建设，数字资源有增长空间
- 比例为 5:0.5:2 → 学校以传统教材为主，数字资源是薄弱环节（机会）
- 连续3年数字资源采购额增长 → 学校在持续加大数字化投入（强信号）
```

### 竞争分析

从采购中标信息中提取：
- 哪些供应商中标（竞品识别）
- 中标金额区间（价格参考）
- 采购方式偏好（公开招标/竞争性磋商/单一来源）

## 利益相关者映射框架

### 角色分类

| 角色 | 职能 | 在采购中的角色 | 对接优先级 |
|------|------|-------------|-----------|
| 分管教学副校长 | 统筹教学工作和资源建设 | **Economic Buyer**（经济买家）— 最终审批预算和方案 | 高（通过引荐） |
| 教务处长 | 管理教学运行和资源调配 | **Decision Influencer**（决策影响者）— 控制预算分配 | 高 |
| 教材科/教学建设科负责人 | 日常教材管理和教学建设 | **日常对接人** — 最容易约到，信息入口 | 最高 |
| 二级学院院长 | 具体专业建设和需求提出 | **Champion候选**（拥护者）— 推荐产品 | 高 |
| 教研室主任/骨干教师 | 具体课程和教材选用 | **End User**（最终用户）— 使用反馈 | 中 |
| 信息中心主任 | 信息化项目归口管理 | **Gatekeeper**（把关者）— 数字资源项目需经其审核 | 中高 |
| 采购部门负责人 | 组织招标和采购流程 | **Paper Process**（流程把关）— 审核资质和组织招标 | 中 |
| 财务处长 | 预算管理和资金拨付 | **Budget Holder**（预算持有者）— 控制资金 | 低（通过其他角色间接影响） |

### 预映射信息来源

| 角色 | 信息来源 |
|------|---------|
| 分管教学副校长 | 学校官网"现任领导"→ 分管教学的校领导 |
| 教务处长 | 学校官网"机构设置" → 教务处 → 处室领导 |
| 教材科负责人 | 教务处网站 → 机构设置 → 教材科 |
| 二级学院院长 | 学校官网"机构设置" → 各二级学院 |
| 信息中心主任 | 学校官网"机构设置" → 信息中心/网络中心 |

### 不同角色的价值主张

| 角色 | 他们关心什么 | 沟通重点 |
|------|------------|---------|
| 分管教学副校长 | 学校战略定位、教学评估结果、双高建设指标 | "我们的方案能帮助学校在XX评估中达到XX指标" |
| 教务处长 | 管理效率、资源统筹、质量管控、教学运行稳定 | "我们的平台能将教材管理效率提升XX%，覆盖XX个专业" |
| 二级学院院长 | 专业建设、课程改革、学生就业率、技能竞赛 | "我们为XX专业定制了XX套资源，学生考证通过率提升XX%" |
| 教研室主任/骨干教师 | 教学减负、课堂效果、学生参与度、备课效率 | "用我们的配套资源，备课时间减少XX%，学生满意度提升XX%" |
| 信息中心主任 | 技术兼容性、数据安全、系统稳定性、运维成本 | "我们的平台支持XX标准，可与现有系统无缝对接" |

## 高职教育决策链条特征

### 教材选用决策链

```
任课教师/教研室主任（提出选用建议）
    ↓
二级学院教学委员会（初审）
    ↓
教务处汇总审核
    ↓
教材选用委员会评审（分管副校长、教务处长、各学院院长、教师代表、行业专家）
    ↓
院长办公会审定
    ↓
采购部门组织采购
```

### 数字资源/信息化项目决策链

```
二级学院/教务处（提出需求）
    ↓
信息中心归口管理审核（技术方案、安全评估）
    ↓
网络安全与信息化领导小组审批（校长/分管副校长、信息中心、教务处、财务处）
    ↓
采购部门组织采购
```

### 关键洞察

1. **教材选用是集体决策**：不是一个人说了算，需要多角色覆盖
2. **数字资源涉及归口管理**：不同类型产品需要找对归口部门（教务处 vs 信息中心 vs 图书馆）
3. **资金来源多元化**：中央奖补、省级配套、学校自筹、社会资本（产教融合），不同资金来源审批流程不同
