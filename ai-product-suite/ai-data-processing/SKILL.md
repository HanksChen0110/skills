# AI产品数据处理

> 工作流位置：需求挖掘 → 用户调研 → 需求分析 → 竞品分析 → 技术选型 → **数据处理** → AI PRD → 评估体系 → Badcase治理

## 数据全流程

```
数据来源识别 → 数据加载 → 数据清洗 → 数据变换
→ 元数据提取 → 分块策略 → 向量化 → 存入向量数据库
→ 用于RAG检索 / 模型微调
```

## 分块策略速查

| 策略 | 说明 | 推荐场景 |
|------|------|---------|
| 固定大小 | 按字数切割 | 快速实现 |
| 语义分块 | 按段落/章节切割 | **推荐默认** |
| 句子滑窗 | 相邻块有重叠 | 信息连续性强 |
| 父子分块 | 小块检索+大块上下文 | 平衡精度与完整性 |

**分块大小**：推荐300-800字，设置10-20%重叠。太小(<100字)语义不完整，太大(>1000字)引入噪声。

## 检索方式对比

| 方式 | 优势 | 劣势 |
|------|------|------|
| 向量检索 | 处理语义相近 | 关键词精确匹配弱 |
| 关键词检索 | 精确匹配 | 无法理解语义 |
| **混合检索** | **兼顾精度和语义** | 需调权重 |

## 召回率优化方向

```
1. 换更好的Embedding模型（参考MTEB排行榜）
2. 优化分块策略和元数据
3. 使用ReRank重排（粗排→精排）
4. Query改写（理解真实意图）
5. 调整TopK参数
```

## PDF处理三类型

| 类型 | 处理方式 |
|------|---------|
| 机器生成PDF | 直接解析文本层 |
| 带OCR扫描文档 | 解析+校验OCR质量 |
| 纯图片PDF | 先跑OCR再提取 |

## 元数据提取要点

必须提取：文件名、类型、时间、章节标题、来源URL/ID、关键词标签。元数据可用于过滤检索范围，大幅提升召回精度。

## 输出物

- 标准化数据集
- 分块策略文档
- 向量数据库
