# AI产品需求挖掘完整方法论

> 来源：[[AI产品需求挖掘]]
> 工作流位置：**需求挖掘** → 用户调研 → 需求分析 → 竞品分析 → 技术选型 → 数据处理 → AI PRD → 评估体系 → Badcase治理

---

## 一、需求来源全景

需求挖掘的核心是**系统性地捕获所有有价值的信号**，而不只是被动接收。主要包含两大块：

### 内部需求
| 来源 | 说明 |
|------|------|
| 公司战略 | 基于战略目标自发演进的产品需求（如年度AI化转型目标） |
| 市场调研&竞品分析 | 通过分析市场和竞品发现的机会点 |
| 数据分析 | 从产品数据中识别出的用户行为异常或瓶颈 |
| 创新 | PM主动洞察新技术、新趋势产生的需求 |

### 外部需求
| 来源 | 说明 |
|------|------|
| 客户差异化需求 | B端客户基于自身业务场景提出的定制需求 |
| 用户反馈 | 通过问卷、访谈、工单等渠道收集的用户意见 |

> **关键认知**：用户往往倾向于直接给出"解决方案"，而非描述"问题本身"。PM需要穿透方案，挖掘背后的真实痛点。

## 二、问题思考框架（PM视角）

### 2.1 发现/挖掘问题

| 维度 | 方法 | 说明 |
|------|------|------|
| **差距感知** | 标杆对比 | 与最优状态对比，识别隐性痛点 |
| **差距感知** | 异常监测 | 从数据杂音中捕获系统失调信号 |
| **系统透视** | 冰山模型 | 从表象事件下钻到深层结构与心智模型 |
| **系统透视** | 瓶颈识别（TOC） | 定位限制系统产出的核心约束点 |
| **矛盾动力学** | 张力识别 | 识别系统中互相冲突的要素（如效率 vs 质量） |
| **价值校验** | 移情映射 | 站在不同利益相关者视角定义问题优先级 |
| **价值校验** | 损益评估 | 基于系统价值损失，判断问题的真实量级 |

**优先级决策**：利用 ICE（影响力×信心×易实现性）或象限法，确认当前矛盾是否值得被解决。

## 三、需求池管理

### 3.1 需求基本信息字段

```
- 所属模块
- 所属业务线
- 需求来源（用户反馈 / 竞品 / 数据 / 内部提出）
- 创建人 & 指派人
- 创建时间
- 优先级（P0 / P1 / P2）
```

### 3.2 需求内容字段

```
- 需求背景：当前存在什么问题，为什么要做
- 需求价值：解决后对用户/业务/公司的价值
- 需求状态：待评估 / 评估中 / 排期中 / 开发中 / 已上线 / 已关闭
```

### 3.3 解决方案字段

```
- 方案描述
- 技术可行性初判
- 预估工期
- 验收标准
```

## 四、需求优先级排序方法

### 4.1 KANO 模型

将需求按用户满意度影响分为五类：

| 类型 | 定义 | AI产品举例 |
|------|------|-----------|
| **必备型（M）** | 不满足则用户极不满，满足了也不加分 | 回答不胡说八道（基础准确性） |
| **期望型（O）** | 满足则满意，不满足则不满 | 回答速度快、内容相关 |
| **魅力型（A）** | 超预期功能，带来惊喜 | 主动追问澄清用户意图 |
| **无差异型（I）** | 有无均不影响满意度 | 某些装饰性功能 |
| **反向型（R）** | 满足反而降低满意度 | 强制弹出广告 |

> **AI产品特别注意**：准确性和安全性属于必备型，任何幻觉和有害输出都是严重缺陷。

### 4.2 四象限法则（ICE变体）

```
重要且紧急 → 立即做（P0）
重要不紧急 → 计划做（P1）
紧急不重要 → 委托或简化处理（P2）
不重要不紧急 → 搁置或关闭
```

## 五、AI场景的需求挖掘要点

### 5.1 判断是否真的需要 AI

在挖掘到需求后，首先要回答：**这个需求用AI解决是最优解吗？**

```
1. 传统技术（规则/NLP/搜索）能否解决？
2. 现有数据量是否足够支撑AI能力？
3. 对错误的容忍度如何？（AI有不确定性）
4. 成本预算是否支撑AI方案？
```

### 5.2 挖掘 AI 需求的关键问题

**从场景出发：**
- 这个业务流程中，哪个环节最耗时/最容易出错/最依赖人工经验？
- 用户在哪个节点最容易流失或表达不满？

**从数据出发：**
- 目前有哪些数据可以被AI利用？
- 数据的质量和数量是否达到AI落地门槛？

**从竞争出发：**
- 竞品是否已有类似AI能力？我们的差异化在哪？

### 5.3 AI 需求的典型来源场景

| 场景 | 典型需求 | 价值方向 |
|------|---------|---------|
| 客服/问答 | 智能问答机器人替代人工 | 降本增效 |
| 内容生产 | 批量生成文案/报告 | 提升效率 |
| 决策辅助 | 数据分析报告自动生成 | 提升质量 |
| 流程自动化 | 审核、分类、抽取信息 | 降本增效 |
| 个性化推荐 | 基于用户行为的动态推荐 | 体验提升 |

## 六、实战示例

**背景**：公司战略要求用AI为其他业线赋能，需要找业务切入点。

**挖掘过程：**
1. 内部咨询发现：供应商对平台产品的使用咨询量很高
2. 人工客服介入成本大，用户满意度低
3. 判断这是一个**高频、有数据、容忍一定错误**的场景 → 适合AI
4. 需求定义：**供应商智能问答系统**，目标是降低人工咨询量、提升回复满意度

**价值验证：**
- 用户侧：提升满意度
- 公司侧：减少人工成本 + 完成AI探索里程碑
- 自身侧：有项目成果，提升影响力

## 七、面试常见问题

**Q：AI产品需求从哪来？**
> 回答框架：内部（战略/数据/创新）+ 外部（用户反馈/B端定制）+ 主动挖掘（差距感知/系统透视）

**Q：如何判断一个需求是否适合用AI解决？**
> 回答框架：传统技术能否解决 → 数据量是否足够 → 错误容忍度 → 成本预算

**Q：需求优先级怎么排？**
> 回答框架：KANO模型分类 → ICE评分 → 四象限法则 → 资源约束下的最终排序
