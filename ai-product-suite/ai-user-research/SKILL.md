# AI产品用户调研

> 工作流位置：需求挖掘 → **用户调研** → 需求分析 → 竞品分析 → 技术选型 → 数据处理 → AI PRD → 评估体系 → Badcase治理

## 核心流程

```
1. 明确调研目标（验证什么假设？结论如何影响决策？）
2. 选择调研方法（问卷/访谈/观察/数据分析）
3. 设计调研工具（问卷/提纲）
4. 选择样本人群（代表性 + 足够样本量）
5. 执行调研（差异化渠道策略）
6. 数据分析（定量+定性）
7. 输出调研结论
```

## 方法选择速查

| 方法 | 适用场景 | 样本量 |
|------|---------|--------|
| 问卷调查 | 量化验证、大样本 | 50人+ |
| 1v1访谈 | 深度挖掘、质性研究 | 5-10人 |
| 焦点小组 | 群体讨论、创意收集 | 6-8人/组 |
| 数据分析 | 行为数据挖掘 | 全量 |

## 问卷设计四原则

1. **用户简单选择**：尽量选择题，减少填写负担
2. **同一问题多角度设问**：交叉验证防敷衍
3. **避免诱导性提问**：不能预设答案
4. **漏斗结构**：从宽泛到具体

## 访谈核心技巧

- **开放式问题**："上次使用这个功能是什么情况？" 而非 "好用吗？"
- **5 Why追问**：从表面需求追到真实痛点
- **沉默是黄金**：用户说完后保持2-3秒沉默

## AI产品调研特别注意

> AI有不确定性，调研时需额外关注用户对"失败情况"的容忍边界。

关键问题：
- 当AI回答错误时，你能接受的频率是？
- 你更在意准确性还是速度？
- 什么情况下你会放弃使用AI转向人工？

## 输出物

- 调研问卷/访谈提纲
- 调研结论报告（核心发现 + 建议）
- 用户画像
