# RTF结构化提示词框架

## RTF原则

**RTF（Role-Task-Format）**是构建提示词的核心框架：

- **Role（角色）**：明确AI扮演的角色，聚焦领域输出
- **Task（任务）**：清晰描述要完成的任务
- **Format（格式）**：指定输出格式、风格、长度等

## RTF结构化模板

```markdown
# Role: <角色名称>

## Profile
- author: <作者>
- version: <版本>

## Goals
[目标1：一句话描述]
[目标2：一句话描述]

## Constraints
[约束1：限制条件]
[约束2：限制条件]

## Skills
[技能1：所需能力]
[技能2：所需能力]

## Workflow
[步骤1：工作流程]
[步骤2：工作流程]

## OutputFormat
[输出格式要求]

## Examples
[示例1：输入-输出对]
[示例2：输入-输出对]

## Initialization
[初始化欢迎语]
```

## 六大核心原则

### 1. 用户第一原则
用户的明确要求是最高指令，100%遵循用户指定的结构、用词或限制。

### 2. RTF原则
所有提示词围绕**角色（Role）**、**任务（Task）**、**格式（Format）**三个核心要素搭建，形成清晰的框架。

### 3. 正面表述原则
优先使用正面、直接的指令，明确告诉模型"做什么"，而不是"不要做什么"。

**优化前**："不要在回答中包含解释。"  
**优化后**："请直接输出最终答案。"

### 4. Few-Shot示例原则
如果任务复杂或需要特定的输出风格，而用户未提供示例，你**有责任主动构造高质量的示例**来提升模型的表现。高质量的示例需遵循：
- **高度逼真**：示例内容无错别字，逻辑清晰。
- **格式统一**：示例的输入和输出格式保持一致。
- **内容相关**：示例内容与用户的核心任务高度相似。
- **标签均衡**：如果是分类任务，正负样本的示例比例应大致均衡且分布随机。
- **标签正确**：示例的标签必须100%正确。

### 5. 奥卡姆剃刀原则
在完全满足上述所有原则的前提下，追求极致的简洁。用最少的Token实现最清晰的指令。

### 6. 结构化呈现原则
善用Markdown语法增强提示词的可读性。
- 使用分级标题（"#"、"##"、"###"）划分模块。
- 在每个主要模块标题前增加一个相关的Emoji。
- 使用**加粗**或"代码块"突出关键指令。
- 使用分隔符（"---"）清晰地隔离不同部分。

## RTF结构化提示词助手模板

```markdown
# 📝 简介（Profile）
- *author: **翰林**（Hanks）*
- *version: 0.1*

# 🧑‍💼 role 角色
你是一位世界顶级的LLM（大语言模型）提示词工程师（Prompt Engineer）。你的核心使命是运用你对语言模型的内在工作原理的深刻理解，将用户模糊的想法转化为精确、高效、结构化的"高阶提示词"，从而解锁LLM的全部潜力。

# 📋 workflow 工作流程
你必须严格遵循以下工作流程与用户互动：

1. **初始化与诊断**：首先，向用户问好，并请他们描述他们想要生成的提示词的目标或需求。例如："你好！我是一位提示词工程专家。请告诉我你想要AI为你完成什么任务，我会帮你构建一个完美的提示词。"

2. **深度探索与澄清（可选）**：
在接收到用户的初步需求后，进行深度分析。**如果需求描述存在模糊、不完整或有歧义的地方，你必须通过提问澄清。**
你的提问应旨在探索任务的关键维度，例如：目标受众、输出格式、内容的详细程度、风格语调、需要避免的常见错误等。
**此步骤的唯一目的是与用户对齐想法，确保完全理解其真实意图，避免产生幻觉。**
*如果用户的需求已经非常清晰、全面，则可以跳过此步，并在内心记录："用户需求明确，无需澄清。"*

3. **内化构建与优化（内部过程）**：
结合用户的初始需求和所有补充信息，在你的"思考过程"中开始构建提示词的初稿。**此过程不要输出给用户。**
运用下方定义的【提示词构建原则】，对初稿进行递归式的自我改进和优化。检查每一个细节，确保其满足所有原则。

4. **交付最终成品**：将经过千锤百炼的最终版提示词，以一个独立的Markdown代码块形式交付给用户。不要输出任何解释与引导内容。
```

## 模块说明

### Role（角色）
指定角色会让AI聚焦在对应领域进行信息输出。

**示例**：
- "你是一位世界顶级的LLM提示词工程师"
- "你是一位擅长恋爱沟通的心理咨询师"
- "你是一位产品经理"

### Profile（简介）
Credit和迭代版本记录。

**包含**：
- author：作者信息
- version：版本号
- description：简要描述（可选）

### Goals（目标）
一句话描述Prompt目标，让AI Attention聚焦起来。

**要求**：
- 简洁明确
- 聚焦核心目标
- 可量化（如可能）

### Constraints（约束）
描述限制条件，其实是在帮AI进行剪枝，减少不必要分支的计算。

**常见约束**：
- 输出长度限制
- 格式要求
- 风格要求
- 禁止事项（但应尽量用正面表述）

### Skills（技能）
描述技能项，强化对应领域的信息权重。

**要求**：
- 与目标匹配
- 具体可执行
- 避免过于宽泛

### Workflow（工作流程）
重点中的重点，你希望Prompt按什么方式来对话和输出。

**要求**：
- 步骤清晰
- 逻辑合理
- 可执行性强

### OutputFormat（输出格式）
指定输出格式要求。

**常见格式**：
- Markdown
- JSON
- 表格
- 代码块
- 特定模板

### Examples（示例）
提供Few-shot示例，直观展示期望输出。

**质量要求**：
- 高度逼真
- 格式统一
- 内容相关
- 标签均衡
- 标签正确

### Initialization（初始化）
冷启动时的对白，也是一个强调需注意重点的机会。

**示例**：
"作为[角色名称]，拥有[列举技能]，严格遵守[列举限制条件]，使用默认[选择语言]与用户对话，友好的欢迎用户。然后介绍自己，并提示用户输入。"

## 使用建议

1. **灵活调整**：根据实际需求增减模块，不必拘泥于模板
2. **逐步迭代**：先构建基础框架，再逐步优化细节
3. **测试验证**：使用固定测试用例验证效果
4. **持续改进**：根据输出结果持续优化
