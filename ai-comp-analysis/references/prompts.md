# AI 竞品分析专用提示词模板

为了获取高质量的一手信息并对抗二手噪音，请在数据采集阶段使用以下提示词模板。

## 1. 架构深度探测 (针对 Kimi/Perplexity)
**场景**：当你需要了解竞品的技术底座和工作流编排时。

```markdown
# Role: 资深 AI 产品架构师
# Task: 深度拆解 [竞品名称] 的技术实现路径

请基于实时检索，帮我分析 [竞品名称] 的以下维度：
1. 模型策略：它是调用闭源 API（如 GPT/Claude），还是基于开源模型（如 Llama/DeepSeek）微调？有无自研模型证据？
2. 编排模式：它更倾向于单 Agent 还是多 Agent 协作？是否有明显的工作流（Workflow）编排痕迹？
3. 知识库实现：它是如何处理 RAG 的？是否有专门的向量数据库选型信息？
4. 核心差异点：相对于通用模型，它在“垂直化”上做了哪些具体的参数微调或知识注入？
```

## 2. 真实口碑与 Badcase 挖掘
**场景**：当你需要评估竞品的“不确定性风险”和用户真实痛点时。

```markdown
# Role: 资深用户研究员
# Task: 挖掘 [竞品名称] 的真实用户负面反馈与体验盲区

请检索社交媒体（X, Reddit, 小红书, 即刻）及专业社区，总结 [竞品名称] 的以下信息：
1. 典型 Badcase：用户反馈最多的“一本正经胡说八道”的场景有哪些？
2. 采纳率瓶颈：用户在什么情况下会觉得 AI 建议不可用，必须手动进行大幅度修改？
3. 傲慢度检测：是否有用户抱怨产品的功能设计“强加了某种工作流”而不够灵活？
4. 稳定性问题：是否存在严重的响应时延或服务中断记录？
```

## 3. 数据飞轮进化评估
**场景**：当你需要判断竞品是否有持续进化的潜力时。

```markdown
# Role: AI 商业策略顾问
# Task: 评估 [竞品名称] 的数据飞轮成熟度

请分析以下维度：
1. 反馈机制：产品界面上是否有明显的、低成本的用户反馈入口（点赞、纠错、采纳）？
2. 进化证据：官方是否有过基于用户反馈进行模型迭代的公开案例或版本记录？
3. 数据资产：它通过当前场景沉淀了哪些“难以被替代”的专有数据资产？
```
