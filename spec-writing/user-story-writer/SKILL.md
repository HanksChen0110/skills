# 用户故事生成技能

使用本技能将需求分析结果转成结构化用户故事。

## 核心目标

- 用用户视角表达需求意图。
- 为功能设计、验收标准和需规提供用户场景输入。
- 保证故事与角色、场景和业务目标一致。

## 默认结构

每组用户故事至少说明：
- 角色
- 场景
- 目标
- 触发条件
- 成功结果
- 关键约束

## 工作流

1. 先确认角色和场景。
2. 将需求拆成独立用户目标。
3. 用统一故事格式表达。
4. 补充异常场景和边界条件。
5. 如有需要，再衔接验收口径。

## 输出建议

- 用户故事集
- 用户故事分组表
- 故事与角色映射
- 故事与功能模块映射

## 禁止事项

- 不得只写一句空泛故事而无场景和目标。
- 不得让用户故事脱离真实业务流程。
- 不得把实现细节混入用户故事正文。
