## 增长认知（重要）

- **增长公式**：`增长 = 内容质量 × 信任密度 × 算法理解 × 用户匹配 × 时机`
- **信任积累路径**：第1次看不懂 → 第3次有点意思 → 第7次稳定靠谱 → 第12次默认相信
- **重要心态**：前10次曝光无增长是正常的，不是失败，是信任库存积累期
- **爆发前兆**：收藏率>点赞率、有人复述你的概念、旧内容复活、粉丝无原因增长

## 步骤1：选题 - 找到值得写的内容

### 核心原则

1. **价值导向**: 解决读者真实问题或提供新视角
2. **受众明确**: 清楚目标读者是谁，他们关心什么
3. **角度独特**: 避免泛泛而谈，找到差异化切入点
4. **可执行性**: 选题要有足够素材支撑，能写完整
5. **时效性**: 结合热点但不盲从，保持独立思考

### 公众号选题特点

- **话题性**: 选择有讨论价值的话题
- **传播性**: 考虑文章的传播潜力
- **垂直性**: 符合公众号定位和受众需求

### 价值判断公式

写作前用公式自检：
```
商业价值 = 被感知收益 × 立即性 × 可理解性 × 信任
```

- **被感知收益**：读者能否立刻意识到能获得什么？
- **立即性**：能否快速应用/见效？
- **可理解**：门槛多高，是否需要前置知识？
- **信任**：读者凭什么相信我讲的这部分？

**自检问题**：
- [ ] 这篇内容的"被感知收益"够明显吗？
- [ ] 读者能快速用到吗？还是需要长期积累？
- [ ] 门槛是否够低，普通人能理解吗？

### 选题检查清单

- [ ] 这个主题能解决什么问题？
- [ ] 目标读者会感兴趣吗？
- [ ] 我的观点或角度有什么不同？
- [ ] 我有足够的素材和案例吗？
- [ ] 这个主题值得花时间深入吗？

## 步骤2：大纲 - 构建清晰的文章骨架

### 核心原则

1. **逻辑递进**: 从问题到方案，从现象到本质
2. **层次分明**: 主论点-分论点-论据，三级结构清晰
3. **重点突出**: 核心观点占60%篇幅，次要内容占40%
4. **开头抓人**: 第一段要能抓住注意力，提出问题或场景
5. **结尾升华**: 总结要点，给出行动建议或启发思考

### 公众号大纲模板

```
H1: 主标题（核心观点）
  H2: 引言（问题/场景）
  H2: 第一部分（核心论点1）
    H3: 分论点1.1
    H3: 分论点1.2
  H2: 第二部分（核心论点2）
    H3: 分论点2.1
    H3: 分论点2.2
  H2: 总结（行动建议/思考）
```

### 大纲检查清单

- [ ] 每个H2是否围绕主论点展开？
- [ ] 逻辑顺序是否合理？
- [ ] 重点内容是否突出？
- [ ] 是否有冗余或缺失的部分？

## 步骤3：正文 - 用文字传递价值

### 核心原则

1. **开门见山**: 第一段直接点题，不要绕弯子
2. **案例支撑**: 每个观点至少配1个具体案例或数据
3. **语言简洁**: 删除冗余词汇，一句话说清楚一件事
4. **节奏把控**: 长短句结合，段落3-5行为宜
5. **情感共鸣**: 用故事和场景让读者产生代入感

### 正文写作技巧

- 用"你"而不是"我们"，增强对话感
- 用具体数字代替模糊表述（"80%"而非"大多数"）
- 用对比突出观点（之前vs之后，错误vs正确）
- 用问句引导思考，增加互动感
- 用金句总结段落，便于记忆和传播
- **开头3秒**：痛点/对比/反常识（抓住注意力）
- **首段无术语**（降低阅读门槛）

### 入口设计技巧

**核心原则**：不改变深度，只改变入口

| 旧写法 | 新写法 | 效果 |
|-------|-------|------|
| 费曼学习法如何提升理解力 | 为什么90%的人"以为自己学会了" | 从原理→痛点 |
| 讲清楚AI思维模型 | 先击中读者的AI使用误区 | 从表达→撞击 |
| 完整体系输出 | 从一个认知冲突开始 | 从终局→切入 |

**写作公式**：
```
【你为什么一直误解】 → 【这个误解为何合理】 → 【撞击点】 → 【比喻/模型】 → 【新理解】
```

**比喻的新角色**：从"展示理解"变成"解锁工具"

**案例**：
- 旧：「费曼学习法是通过输出暴露理解漏洞，就像编译器检查代码……」
- 新：「为什么你明明"看懂了"，却一周后完全讲不出来？费曼学习法其实像编译代码——只有运行时报错你才知道哪里没懂。」

### 金句提炼

**金句特点**：
- 简洁有力，一句话说清楚核心观点
- 易于记忆和传播
- 能够引发共鸣

**金句位置**：
- 段落结尾
- 章节总结
- 文章结尾

### 正文检查清单

- [ ] 每个段落是否围绕一个中心？
- [ ] 案例是否贴切且有说服力？
- [ ] 语言是否简洁易懂？
- [ ] 是否有金句可以提炼？

## 步骤4：标题 - 抓住第一眼注意力

### 核心原则

1. **价值明确**: 一眼看出能获得什么（知识/方法/启发）
2. **数字具体**: 用数字增强可信度（"3个方法"而非"几个方法"）
3. **情绪共鸣**: 击中读者痛点或爽点
4. **悬念适度**: 引发好奇但不故弄玄虚
5. **长度适中**: 公众号标题20-30字最佳

### 标题公式

- **数字型**: "X个方法/技巧/原则..."
- **对比型**: "从X到Y，我学会了..."
- **问题型**: "为什么X总是Y？"
- **场景型**: "当你X时，记住Y"
- **结果型**: "做了X，我得到了Y"

### 标题检查清单

- [ ] 读者能一眼看出价值吗？
- [ ] 是否足够具体和可信？
- [ ] 能引发点击欲望吗？
- [ ] 是否与正文内容匹配？

## 步骤5：排版 - 公众号适配

### 核心原则

1. **三级标题**: 只用 H1/H2/H3
2. **段落分割**: 用'---', 上下必须空行
3. **有序列表**: 数字后无空格 ('1.项目')
4. **行高 1.8**: 书籍标准
5. **克制配色**: 灰色管结构，蓝色管交互，黑色管内容

### 公众号排版规范

- **段落间距**: 段落之间用空行分隔，重要段落用'---'分隔
- **加粗强调**: 关键信息用加粗突出
- **Obsidian 加粗**: 加粗的内容需要和前后文用一个空格隔开（即开头的 `**` 前面和结尾的 `**` 后面都需要一个空格），标点保留在加粗符号内。例如： ` **但扎心的是：一片空白。** `
- **引用标注**: 重要观点用引用格式
- **列表格式**: 有序列表和无序列表规范使用
- **插图位点**: 在关键位置预留插图位点
- **图片引用格式**: `![标题](url)` 中方括号内不带扩展名（如 `![技术模型](url.png)` 而非 `![技术模型.png](url.png)`）

### 自动排版处理项（必做）

- **图片标题去扩展名**: 将 `![xxx.png](url.png)` 转为 `![xxx](url.png)`，仅处理方括号内文本

### 排版检查清单

- [ ] 标题层级是否清晰？
- [ ] 段落是否用'---'分隔？
- [ ] 列表格式是否正确？
- [ ] 图片引用方括号内无扩展名（`![标题]` 而非 `![标题.png]`）
- [ ] 整体视觉是否舒适？

## 反模式（NEVER Do）

### ❌ 反模式1：选题过于宽泛

**错误做法**: "如何提高工作效率"（太宽泛）
**正确做法**: "3个时间管理技巧，让我的工作效率提升80%"

### ❌ 反模式2：大纲逻辑混乱

**错误做法**: 想到哪写到哪，没有清晰结构
**正确做法**: 先构建三级结构（主论点-分论点-论据）

### ❌ 反模式3：正文缺乏案例

**错误做法**: 只有观点，没有具体案例支撑
**正确做法**: 每个观点至少配1个具体案例或数据

### ❌ 反模式4：标题故弄玄虚

**错误做法**: "震惊！这个方法竟然..."（过度营销）
**正确做法**: "3个方法，解决你的XX问题"（价值明确）

### ❌ 反模式5：排版不规范

**错误做法**: 标题层级混乱，段落不分隔
**正确做法**: 遵循三级标题、段落分隔、列表格式规范

### ❌ 反模式6：忽略CTA设计

**错误做法**: 文章结尾没有行动号召
**正确做法**: 设计明确的CTA，引导读者行动

## 2. 详细步骤指令

### Step 1: 选题确认 (Topic Validation)
- **动作**: 询问用户想写什么，并基于"清醒边界"人设进行 **选题审计**。
- **审计标准**:
  - 是否太泛？(e.g., "AI怎么用" -> 拒; "DeepSeek vs GPT-4o 的B端落地对比" -> 过)
  - 是否有独特的"我"的视角？
- **输出**: 确认后的选题 + 核心切入点。

### Step 2: 大纲构建 (Outline)
- **动作**: 生成三级大纲 (H1/H2/H3)。
- **模板**:
  ```markdown
  H1: [暂定标题]
    H2: 引言 (SCQA架构: 情境-冲突-问题-答案)
    H2: 核心论点1 (What)
    H2: 核心论点2 (Why - 深度分析)
    H2: 核心论点3 (How - 实操落地)
    H2: 总结与升华 (CTA)
  ```
- **约束**: 必须引用 `references/methodology.md` 中的逻辑原则。

### Step 3: 正文撰写 (Drafting)
- **动作**: 逐章撰写正文。
- **约束**:
  - **去噪**: 删除所有"众所周知"的铺垫。
  - **案例**: 每个论点必须配 1 个具体案例/数据。
  - **金句**: 每段结尾尝试提炼金句。
  - **语言**: 使用"你"而非"我们"，增强对话感。

### Step 4: 质量评估 (Audit & Refine Loop) [CRITICAL]
**⚠️ 正文写完后，必须立即自动执行此步骤，无需用户请求。**

1.  **自动打分**: 基于 `references/methodology.md` 中的 **0-100分标准** 进行 4 维打分：
    - 认知密度 (40分)
    - 实操价值 (30分)
    - 逻辑结构 (15分)
    - 表达风格 (15分)

2.  **判断逻辑**:
    - **IF 总分 < 80**:
      - ❌ **拒绝发布**。
      - 🔴 **指出问题**: "扣分点：[具体原因]"。
      - 🔄 **自动修正**: "正在优化... (针对扣分点重写相关段落)"。
      - ➡️ **重新评估**: 修正后再次打分，直到 ≥ 80。
    - **IF 总分 ≥ 80**:
      - ✅ **通过**。
      - 显示最终得分雷达图/表格。
      - 进入下一步。

### Step 5: 标题推荐 (Headline Generation)
- **动作**: 基于正文生成 **5 个** 候选标题。
- **要求**: 覆盖不同风格，并给出推荐理由。
- **格式**:
  1. **[数字型]** 标题文案 (推荐指数: ⭐⭐⭐⭐⭐) - 理由...
  2. **[悬念型]** 标题文案 (推荐指数: ⭐⭐⭐⭐) - 理由...
  3. **[痛点型]** 标题文案 (推荐指数: ⭐⭐⭐⭐) - 理由...
  4. **[反直觉型]** 标题文案 (推荐指数: ⭐⭐⭐) - 理由...
  5. **[利益型]** 标题文案 (推荐指数: ⭐⭐⭐) - 理由...
- **交互**: "请回复序号选择标题，或要求重新生成。"

### Step 6: 排版输出 (Formatting)
- **动作**: 输出最终的 Markdown 代码块，方便复制到 Obsidian/公众号编辑器。
- **排版规范 (Strict Rules)**:
  - **三级标题**: 仅用 H2, H3 (H1留给文章主标题)。
  - **列表**: 有序列表 `1.` 后必须空格。
  - **加粗**: `**重点**` (不仅是词，可是短句)。
  - **分割**: 段落间空行，大模块间用 `---`。
  - **图片**: `![图片说明](url)` (去除 `.png/.jpg` 后缀)。
  - **引用**: 关键概念/金句使用 `> 引用格式`。
