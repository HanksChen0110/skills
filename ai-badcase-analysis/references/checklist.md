# Badcase分析检查清单汇总

本文档汇总了badcase分析全流程的所有检查清单，方便快速查阅。

---

## 阶段1：认知建立

### 基础认知检查

- [ ] 是否理解badcase是正常现象（人类也会出错）？
- [ ] 是否明确分析badcase的核心价值（症状诊断、经验积累）？
- [ ] 是否准备好投入10-30分钟进行分析？

---

## 阶段2：现状评估

### 2.1 评测集检查

- [ ] 数据集是否具有完整意义（不是一两条case）？
- [ ] 数量是否足够支撑指标计算稳定（建议100+条）？
- [ ] 抽样情况是否与实际使用场景一致？
- [ ] 数据质量是否可靠（自己标注100条验证，准确率95%+为佳）？

### 2.2 指标设计检查

- [ ] 指标是否体现观测目标（如避免误召回）？
- [ ] 指标口径是否明确（不同条件取数据可能结论不同）？
- [ ] 是否组合多个指标观测（准确率+召回率）？

### 2.3 现状结论检查

- [ ] 算法效果好or不好？（与预期目标对比）
- [ ] 是否需要进一步优化？
- [ ] 优化方向是什么？（优化准确率、扩大召回等）

---

## 阶段3：问题分析

### 3.1 样本选择检查

- [ ] 是否结合现状分析确定了要优化的方面（准确率/召回率）？
- [ ] 样本数量是否足够（建议50-200个）？
- [ ] 是否采用随机抽样（非随机可能导致主干问题不出现在样本中）？

### 3.2 分析方法检查

#### 粗看法检查

- [ ] 是否关注了长度、句式、问句等特征？
- [ ] 是否总结了突出问题？

#### 追溯法检查

- [ ] 是否准备了日志，分析预测每个阶段的结果？
- [ ] 是否去训练样本查找是否含有类似badcase？
- [ ] 是否检查了正负样本比例是否失衡？

#### 解决法检查

- [ ] 是否把问题归纳到对应解决方案上？

### 3.3 问题标记和统计检查

- [ ] 是否对每个case做了关键记录（归因、解决方案）？
- [ ] 词汇和问题是否统一，方便统计占比？
- [ ] 一个问题是否有多个解决方法，是否都标记了？
- [ ] 是否统计了各问题分布情况？
- [ ] 是否识别了重点问题（占比高的问题）？
- [ ] 是否评估了重点问题的解决价值（占比50%的问题，完全解决可提升20%准确率）？

---

## 阶段4：解决方案

### 4.1 解决方案选择检查

#### 样本误导问题检查

- [ ] 是否识别出数据中浅层直白的信息误导了模型？
- [ ] 是否添加了带误导性特征的负样本？

#### 阈值确定检查

- [ ] 是否准备了标注数据用于阈值实验？
- [ ] 是否测试了不同阈值下的准确率和召回率？
- [ ] 是否根据业务场景确定了最佳阈值（搜索对话场景一般0.85+）？

#### 前后处理检查

- [ ] 是否识别了干扰项（看badcase，错误样本会告诉你）？
- [ ] 是否考虑了前处理（能用规则尽量用规则）？
- [ ] 是否考虑了后处理（模型难学习的东西通过后处理修复）？

### 4.2 验证和迭代检查

- [ ] 是否重跑验证集，看修复后效果？
- [ ] 是否回到第一步：现状确认？
- [ ] 是否重新确定是否还要优化？
- [ ] 如需优化，是否继续badcase分析？

### 4.3 换模型时机判断检查

- [ ] 是否能看到当前方案的天花板？
- [ ] 天花板标志：
  - [ ] 很多零散的badcase
  - [ ] 问题不太能总结出来
  - [ ] 总结出来发现分布很均匀，每个只错一两条
- [ ] 是否靠修补提升不了效果了？

---

## 通用注意事项检查

- [ ] 是否跳过了现状评估？（不要发现一个badcase就去修）
- [ ] 样本数量是否足够？（50-100条即可说明问题）
- [ ] 是否采用了随机抽样？（非随机可能导致主干问题不出现在样本中）
- [ ] 是否进行了问题统计？（通过统计占比找到重点问题）
- [ ] 是否优先考虑数据、阈值、前后处理？（大多数问题不在模型）

---

## 快速检查流程

### 首次分析前

1. [ ] 认知建立检查（3项）
2. [ ] 现状评估检查（10项）
3. [ ] 确定优化方向

### 分析过程中

1. [ ] 样本选择检查（3项）
2. [ ] 分析方法检查（根据选择的方法）
3. [ ] 问题标记和统计检查（6项）

### 解决方案实施

1. [ ] 解决方案选择检查（根据问题类型）
2. [ ] 验证和迭代检查（4项）
3. [ ] 如需换模型，进行换模型时机判断检查（4项）

### 持续优化

1. [ ] 回到现状评估
2. [ ] 重复分析流程
3. [ ] 螺旋上升
