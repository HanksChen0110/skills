# 公众号内容写作

## 核心工作流

```
用户输入（话题/素材/Brief/链接/PDF/语音转文字）
  → Step 1: 素材消化 + 内容原型判定（5选1）
  → Step 2: 大纲生成（加载对应原型的reference文件）
  → Step 3: 用户确认大纲
  → Step 4: 正文撰写（遵循原型模板 + voice-guide）
  → Step 5: 自动评分（5维度，100分，≥80分通过）
       ├── < 80分：诊断 + 自动修改 + 重新评分（最多3轮）
       └── ≥ 80分：通过
  → Step 6: 调用 wechat-title skill 生成标题 + 摘要
  → Step 7: 最终排版输出
```

---

## Step 1: 素材消化 + 原型判定

### 素材处理

用户可能提供任何格式的输入。不论格式，都要完成：
1. **提取核心信息**: 这是关于什么的？核心观点/发现是什么？
2. **判断原型**: 属于下表哪种？
3. **确认与用户**: 如果不确定，先问用户再继续

### 5种内容原型

| 原型 | 判断依据 | 加载的reference |
|------|---------|----------------|
| **产品测评** | 围绕具体产品/工具，有实测体验 | `references/archetype-product-review.md` |
| **热点追踪** | 基于行业新闻/事件，时效性强 | `references/archetype-hot-topic.md` |
| **方法论/教程** | 提供可操作步骤/工作流 | `references/archetype-methodology.md` |
| **现象叙事** | 观察趋势/社会现象，有个人思考 | `references/archetype-phenomenon.md` |
| **概念重构** | 重新定义概念，挑战既有认知 | `references/archetype-concept.md` |

**判定后只加载对应的1个reference文件，不要全部加载。**

## Step 2-3: 大纲生成 + 用户确认

加载对应原型的reference文件后，按其中的结构模板生成大纲。大纲格式：

```
## 大纲

**原型**: [产品测评/热点追踪/方法论/现象叙事/概念重构]
**预计字数**: [3000-8000字]

### 开头（100字内）
- 开头类型: [叙事触发/荒诞事实/结论前置]
- 核心钩子: [一句话描述]

### 中段（3-5节）
1. [小节标题] — [核心观点] — [配图/截图需求]
2. [小节标题] — [核心观点] — [配图/截图需求]
3. ...

### 结尾
- 金句方向: [一句话]
- CTA类型: [互动引导/资源分发/行动号召]
```

**必须等用户确认大纲后才进入正文撰写。**

## Step 4: 正文撰写

### 写作声音

阅读 `references/voice-guide.md` 获取完整声音定义。核心要点：

**融合三家**:
- **饼干哥哥的实战感**: 结论前置、Step结构、"你今天就能用"
- **卡兹克的人文温度**: 故意打破、情绪身体记忆、私人视角连接公共议题
- **洛小山的深度拆解**: 数字即信任、自创概念、叙事先于结论

**一句话**: 用实战经验说话、用人文视角观察、用深度拆解证明。

### 开头（前100字）— 3种有效模式

**A. 叙事触发** — 用具体时间/地点/动作开场
> 示例: "故事是这样的。" / "这个世界，终于魔幻到我看不懂的程度了。"

**B. 荒诞事实** — 直接抛出一个让人"？？？"的事实
> 示例: "卧槽？？？" / "不行就润吧，国庆+中秋总共就4天。"

**C. 结论前置** — 直接给出核心判断
> 示例: "废话不多说，先说结论。" / "先看下这个吧。"

**绝对禁止**: "随着AI技术的发展……" / "在当今……" / "本文将探讨……"

### 正文结构

- **3-5个子章节**，每章200-400字
- **短段落**: 每段1-3句（移动端优先）
- **频繁换行**: 制造"呼吸空间"
- **每节必须有**: 一个清晰观点 + 一个具体例子/截图位 + 与读者处境的连接
- **每600-800字**: 一次情绪转折（震惊/自嘲/温情/焦虑）
- **截图标记**: 用 `[图片：描述]` 标注需要配图的位置

### 人称

- **第一人称主导**: "我试了/搭了/发现了……"
- **直接称呼读者**: "你" 而非 "我们"
- **坦诚不确定**: "说实话我也不确定" "这个事儿我也踩过坑"

### 口语化词组（自然使用，不要硬塞）

**转场**: 其实吧、说实话、不过、回到这块、话说回来、有一说一
**判断**: 我觉得、我发现、我自己的感受是、反正我觉得
**坦白**: 说实话我也不确定、不得不承认、我低估了
**情绪**: 太爽了、太离谱了、牛逼、你敢信？？？
**拉近**: 你可能不知道、你有没有遇到过、大家看下效果

### 禁用词（一票否决）

综上所述、值得注意的是、说白了、意味着什么？、本质上、换句话说、不可否认、赋能、抓手、闭环、颗粒度、笔者、本文将、希望对你有所帮助

完整禁用清单见 `references/voice-guide.md`。

### 结尾

- **价值观金句**: 抓住文章精髓的一句可转发的话
- **明确CTA**: 互动引导 / 资源分发 / 行动号召
- **绝不总结**: 结尾是新思考，不是回顾
- **不用**: "综上所述""总而言之""希望对你有所帮助"

## Step 5: 自动评分

### 5维度，100分

| 维度 | 分值 | 评分要点 |
|------|------|---------|
| 认知密度 | 20分 | 信息/字数比；避免说正确的废话；提供真正洞察 |
| **实操价值** | **30分** | 读者今天就能动手；具体步骤/工具点名；学习曲线坦白；可复现性 |
| 结构质量 | 15分 | 开头100字内钩住；逻辑流畅；每节有明确目的；结尾升华 |
| 表达与声音 | 20分 | 第一人称真实感；口语节奏；无AI味词汇；人格一致性 |
| 互动设计 | 15分 | 互动钩子；可分享性；CTA清晰度；"社交货币"价值 |

### 一票否决（无论总分直接不通过）

- 包含任何禁止开头模式（"随着……""在当今……""本文将……"）
- 没有具体产品/工具名（通篇"AI工具"泛称）
- 没有第一人称经验或观点
- 纯信息堆砌无判断

### 评分输出格式

```
## 质检报告

| 维度 | 得分 | 诊断 |
|------|------|------|
| 认知密度 | X/20 | ... |
| 实操价值 | X/30 | ... |
| 结构质量 | X/15 | ... |
| 表达与声音 | X/20 | ... |
| 互动设计 | X/15 | ... |
| **总分** | **X/100** | |

一票否决检查: ✅/❌
```

### 自动修改循环

评分 < 80 时：
1. 定位2个最低维度
2. 输出具体诊断（"实操价值12/30因为第3节没有可执行步骤"）
3. 自动修改相关段落
4. 重新评分
5. 最多循环3次，仍不达标则请求用户人工介入

完整评分标准见 `references/scoring-rubric.md`。

## Step 6: 标题 + 摘要

正文通过评分后，调用 `wechat-title` skill 生成5个候选标题和配套摘要。

如果用户当前会话中未加载 wechat-title skill，则按以下简化流程：
- 生成5个候选标题（遵循 wechat-title 的公式）
- 为Top 3生成摘要
- 提示用户可以调用 `/wechat-title` 获得完整评分

**关键规则**：选定最终标题和配套摘要后，必须在输出文章中**直接嵌入**推荐的标题和摘要。嵌入方式：
- 如果文章使用 YAML frontmatter，在 frontmatter 中增加 `recommend_title` 和 `recommend_digest` 字段
- 如果文章无 frontmatter，在文章最顶部（正文之前）用引用块标注推荐标题和摘要

示例（有 frontmatter 时）：
```yaml
---
title: "文章主标题"
recommend_title: "推荐标题文字"
recommend_digest: "推荐摘要文字"
status: draft
---
```

---

## Step 7: 排版输出

- H1仅用于文章标题，H2/H3用于子章节
- 段落间空行分隔
- 关键洞察加粗或引用块
- 图片位置标记: `[图片：描述]`
- **必须包含推荐标题和摘要**（见 Step 6 关键规则）
- 输出markdown格式，兼容 `baoyu-markdown-to-html` 和 `baoyu-post-to-wechat` skill
