# 公众号标题与摘要生成

## 核心工作流

```
输入：文章内容 / 主题 / 草稿
  → Step 1: 识别内容原型（5种之一）
  → Step 2: 根据原型选择标题公式
  → Step 3: 生成5个候选标题
  → Step 4: 对每个标题7维度评分（满分100）
  → Step 5: 为Top 3标题生成配套摘要
  → Step 6: 输出结果 + 评分明细
  → 用户选择后可迭代优化
```

---

## Step 1: 识别内容原型

根据文章内容判断属于以下哪种原型：

| 原型 | 判断依据 |
|------|---------|
| **产品测评** | 围绕某个具体产品/工具展开，有实测体验 |
| **热点追踪** | 基于行业新闻/事件/发布会，时效性强 |
| **方法论/教程** | 提供可操作的步骤/工作流/最佳实践 |
| **现象叙事** | 观察行业趋势/社会现象，有个人思考 |
| **概念重构** | 重新定义某个概念，挑战既有认知 |

---

## Step 2: 标题公式（每种原型对应一套公式）

### 产品测评类

**公式**: [第一人称动作] + [产品名] + [意外结果/判断]

适用变体：
- "我用 [产品] [时间]，[出乎意料的结论]"
- "[产品名] [版本]，[核心变化] + [我的判断]"
- "[数字]个理由，让我彻底[转向/放弃] [产品A] 换 [产品B]"

高分示例：
- 「我用了一个月Dia浏览器，已经彻底抛弃了Chrome！」— 30,948阅读
- 「花了十万块，这就是我2026还会用的AI大合集！」— 28,156阅读

### 热点追踪类

**公式**: [事件主体] + [核心动作/冲突] + [影响判断]

适用变体：
- "[公司/产品] [刚刚做了什么]：[一句话影响]"
- "[产品名] 发布 — [一个词/短语的震撼评价]"
- "[权威机构] [动作]：[核心冲突]"

高分示例：
- 「OpenAI全新发布文生视频模型Sora - 现实，不存在了」— 100,001阅读
- 「白宫宣布：封杀 Claude」— 78,175阅读
- 「国家超算互联网，上线 DeepSeek」— 100,001阅读

### 方法论/教程类

**公式**: [数字] + [动作动词] + [具体收益/结果]

适用变体：
- "[N]个[方法/工作流]，[直接/轻松][具体结果]"
- "[时间/成本] + [动作]，[结果]"
- "一套[方法/提示词]帮你实现[具体自由]"

高分示例：
- 「15个n8n工作流，直接干掉90%的跨境营销工作」— 39,657阅读
- 「6000字+6个案例：写给普通人的MCP入门指南」— 29,256阅读
- 「一套提示词帮你实现小红书、公众号封面自由」— 27,842阅读

### 现象叙事类

**公式**: [可观察事实/个人经历] + [意外推论/深层真相]

适用变体：
- "[我做了/我花了/我经历了X]，[看到了/发现了Y]"
- "[现象] 背后，[更深层的X]"
- "[事物A]和[事物B]，[意外的联系]"

高分示例：
- 「淘宝上卖9块9的DeepSeek，让我看到了被折叠的魔幻世界」— 100,001阅读
- 「我花499找人上门安装OpenClaw，看到了AI时代最魔幻的一幕」— 100,001阅读
- 「我花10小时写的原创，被AI在10分钟内偷走了。」— 13,001阅读

### 概念重构类

**公式**: [熟悉概念] + [范式转换/重新定义]

适用变体：
- "[概念]是错的。真正的[事物]是[新框架]"
- "[旧理解] → [新理解]"
- "[熟悉词汇]，应该[重新翻译/理解]为[新定义]"

高分示例：
- 「Token，应该翻译成「新智元」」— 34,248阅读
- 「我们，已迈过奇点」— 53,976阅读

## Step 3: 生成5个候选标题

生成时遵循以下硬规则：

### 必须满足

1. **含具体产品/工具名** — 不说"AI工具"，说"Claude Code""DeepSeek""Seedance 2.0"
2. **陈述句优先** — 5个候选中至少3个是陈述句
3. **25-35字甜区** — 每个标题控制在这个区间（±5字可接受）
4. **含数字或具体成果** — 5个候选中至少3个含数字
5. **制造张力/反差** — 预期 vs 结果、成本 vs 收益、简单 vs 强大

### 绝对禁止

- 纯疑问句标题（"如何用AI写文章？"）
- 泛AI关键词无具体产品（"AI工具推荐"）
- 教程式开头（"如何/怎么"开头 — 跨博主验证低表现）
- 标题短于20字（信息量不足）
- "干货""硬核""强烈推荐"等自我标榜词
- 多个感叹号连用（"太炸了！！！"）

### 5个候选的多样性要求

| 候选 | 策略 |
|------|------|
| #1 | 主打公式：最匹配原型的标准公式 |
| #2 | 第一人称变体：强化"我"的视角和体验 |
| #3 | 数字冲击变体：突出数字/成本/时间 |
| #4 | 反差/冲突变体：制造认知反差 |
| #5 | 克制变体：更短、更冷静、更有判断力 |

## Step 4: 7维度评分（满分100）

对每个候选标题逐项评分：

| 维度 | 分值 | 评分标准 |
|------|------|---------|
| **产品具体性** | 15分 | 15分：点名真实产品+版本号；10分：点名产品无版本；5分：仅品类名；0分：无产品 |
| **价值清晰度** | 20分 | 读者能否在2秒内知道"我能得到什么"？20分：一目了然；10分：需要猜测；0分：完全模糊 |
| **情绪触发力** | 10分 | 10分：强好奇/兴奋/紧迫；5分：一般兴趣；0分：无感 |
| **数字/证据感** | 10分 | 10分：含具体数字且有说服力；5分：含数字但较随意；0分：无数字无证据 |
| **长度优化** | 10分 | 10分：25-35字；7分：20-24字或36-40字；3分：其他；0分：<15或>50字 |
| **句式得分** | 15分 | 15分：陈述句+判断力；10分：陈述句普通；5分：感叹句；0分：纯疑问句 |
| **差异化** | 20分 | 在一屏同类AI文章中能否脱颖而出？20分：独特视角；10分：有特色；0分：同质化 |

**及格线**: 70分。低于70分的候选必须替换或修改。

---

## Step 5: 摘要生成

为评分Top 3的标题各生成一条配套摘要。

### 核心原则

摘要是标题的**互补**，不是重复。通过"第二个点击理由"测试：标题让你好奇，摘要让你确定要点。

### 三种摘要策略

| 策略 | 适用场景 | 示例模式 |
|------|---------|---------|
| **扩展策略** | 标题给结果，摘要给范围 | 标题："15个n8n工作流干掉90%营销工作" → 摘要："覆盖内容创作、竞品监控、客户跟进等10个场景，附完整配置文件" |
| **补充策略** | 标题给判断，摘要给背景 | 标题："我彻底抛弃了Chrome" → 摘要："一个月深度使用Dia浏览器后的真实感受，附迁移指南" |
| **钩子策略** | 标题给话题，摘要给悬念 | 标题："淘宝9.9的DeepSeek" → 摘要："花9块9买下三个商品，我看到了AI行业最真实的样子" |

### 摘要约束

- **40-80字**（微信摘要显示限制）
- 不重复标题中已有的关键词
- 不用"本文将""本篇介绍"等百科式语言
- 必须包含一个标题中没有的信息点（范围/背景/悬念）

## 迭代优化

用户选择候选后，可请求：
- **微调**: 保持核心不变，调整措辞/长度/语气
- **换方向**: 用不同公式重新生成该位置的候选
- **优化摘要**: 为选定标题重新生成摘要（尝试不同策略）

每次微调后重新评分，确保 ≥70 分。
