## 一、为什么要做用户调研

用户调研是**验证需求真实性**的核心手段，能帮助你：

- **验证假设**：你认为的痛点是否真的存在？
- **挖掘深层需求**：用户表达的需求背后，真正想要的是什么？
- **量化问题规模**：这个问题影响了多少用户？频率有多高？
- **避免方向性错误**：用调研成本代替开发成本提前试错

> **AI 产品特别重要**：AI 有不确定性，调研时需额外关注用户对"失败情况"的容忍边界。

## 二、调研前的准备

### 2.1 明确调研目标

```
- 我想通过调研验证什么假设？
- 我期望从用户那里得到什么答案？
- 调研结论将如何影响产品决策？
```

**示例**：
> 假设：供应商对现有问答机器人不满意，主要原因是答案不准确
> 期望答案：不满意的具体原因分布 + 满意度基线数据

### 2.2 选择调研方法

| 方法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **问卷调查** | 量化验证、大样本 | 成本低、速度快、数据可量化 | 无法深挖，用户可能敷衍 |
| **用户访谈（1v1）** | 深度挖掘、质性研究 | 能追问、获取真实动机 | 耗时、样本量小 |
| **焦点小组** | 群体讨论、创意收集 | 有碰撞，发现意外观点 | 容易出现从众效应 |
| **用户观察** | 验证实际操作行为 | 最真实，发现用户说不出的痛点 | 成本高、难操作 |
| **数据分析** | 行为数据挖掘 | 客观、规模大 | 只知道"做了什么"，不知道"为什么" |

### 2.3 选择样本人群

```
- 样本要具有代表性：覆盖核心用户群体的不同类型
- B端产品：区分"决策者"和"使用者"，两者诉求不同
- C端产品：按画像分层（新用户/老用户/高频/流失用户）
- 样本量：定性访谈 5-10 人，定量问卷 50 人以上才有参考价值
```

## 三、问卷设计原则

### 3.1 核心原则

1. **以用户简单选择为原则**：尽量用选择题，减少填写负担
2. **同一问题多角度设问**：交叉验证，避免用户随意勾选
3. **避免诱导性提问**：不能问"你觉得我们产品是不是很好？"
4. **漏斗结构**：从宽泛到具体，先建立信任再问敏感问题

### 3.2 真实性验证技巧

```
针对同一核心问题，用不同表达设问：

问题A：你觉得机器人回答准确吗？（直接问）
问题B：上周你有几次没有得到满意的回答？（行为量化）
问题C：你是否曾因为回答不准确而转向人工？（结果验证）

三个问题答案逻辑一致 → 可信
三个问题答案矛盾   → 需进一步访谈核实
```

### 3.3 示例题目设计（智能问答场景）

```
1. 现有机器人回答准确率，满意度如何？（1-5 分）
2. 你最希望机器人在哪方面提升？（多选）
   □ 回答准确性  □ 回答速度  □ 理解我的问题  □ 回答完整性
3. 机器人的回复你是否满意？（满意 / 一般 / 不满意）
4. 不满意的具体原因？（可选填）
5. 遇到回答不好时，你会怎么做？
   □ 换个方式再问  □ 转人工客服  □ 放弃咨询  □ 其他
```

## 四、沟通策略（差异化渠道）

| 用户类型 | 渠道策略 |
|---------|---------|
| **G端（政府/大客户）** | 联系驻地运营，通过正式渠道邀请参与 |
| **B端企业用户** | 销售/CSM 协助，借助业务关系建立信任 |
| **C端普通用户** | 产品内弹窗、邮件邀请、社区发帖 |
| **核心种子用户** | 建立 1v1 关系，定期访谈，给予特权/福利作为回报 |

**提升参与率**：明确告知调研目的和用时（"只需 3 分钟"）+ 提供激励（积分、优惠券）

## 五、访谈技巧

### 5.1 访谈前准备

```
- 确定访谈提纲（核心问题 5-8 个）
- 确定目标：本次最想验证的 1-2 个假设
- 准备追问清单：每个问题的深挖方向
```

### 5.2 关键技巧

**多用开放式问题：**
```
❌ "你觉得我们的产品好用吗？"（封闭，只能答是/否）
✅ "你上次使用这个功能是什么情况？发生了什么？"（引导讲故事）
```

**追问 5 Why（示例）：**
```
用户："回答太慢了"
→ "慢到什么程度你会觉得不能接受？"
→ "超过 10 秒我就不想等了"
→ "等待时你通常在做什么？"
→ 真实痛点：打断感，而非绝对速度
```

**沉默是黄金**：用户说完后保持 2-3 秒沉默，往往引出更真实的补充。

### 5.3 面试考察点（实战参考）

> **面试官可能问：**
> - 你期望从用户调研中得到什么答案？
> - 你是怎么找到这些访谈用户的？
> - 你怎么判断用户反馈是真的还是假的？
> - 你的数据分析维度是什么？

**回答框架**：调研目标 → 方法选择 → 样本策略 → 数据分析 → 结论应用

## 六、调研数据分析

### 6.1 定量分析维度

```
- 满意度分布：高/中/低满意度用户各占多少
- 痛点分布：不满意原因 Top3
- 用户画像分布：不同类型用户的需求是否一致
- 行为数据交叉：满意度低的用户，行为特征有何共性
```

### 6.2 定性数据处理

```
收集原始表述 → 归类合并相似观点 → 统计频次 → 标记关键引语
→ 识别矛盾点（需进一步核实的信号）
```

### 6.3 结论输出格式

```
1. 调研背景与目的
2. 调研方法与样本说明
3. 核心发现（3-5 条）
4. 用户痛点排序
5. 对产品决策的建议
6. 附录：原始数据/访谈记录摘要
```

## 七、AI产品调研特别注意

### 7.1 必须关注的额外问题

> AI有不确定性，调研时需额外关注用户对"失败情况"的容忍边界。

关键问题：
- 当AI回答错误时，你能接受的频率是？
- 你更在意准确性还是速度？
- 什么情况下你会放弃使用AI转向人工？

### 7.2 调研结果与AI可行性评估

```
调研发现的问题 → 判断AI能否解决 → 估算用户价值

高价值 + AI可解决 → 优先做
高价值 + AI暂不可解决 → 人工兜底或等待技术成熟
低价值 → 搁置
```
